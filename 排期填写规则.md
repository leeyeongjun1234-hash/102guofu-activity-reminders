# 国服活动排期填写规则

这份说明按当前脚本实际逻辑整理。后续新增排期时，优先按这里填；如果新活动不符合这些规则，再改脚本规则。

## 相关文件

- `102国服活动排期表.xlsx`：主排期表。脚本按制表符文本读取，关键是第 2 行日期和日期列里的活动内容。
- `活动与礼包对应关系.xlsx`：活动名到活动 ID/展示名的映射表。
- `daily_reminder.py`：生成 `每日活动提醒.txt`，同时负责活动识别、提醒日期、服务器、时间计算。
- `build_reminders_html.py`：生成 `全部每日提醒.html` 和 `site/index.html`，负责网页展示和部分礼包详情展示。
- `watch_html_auto_update.py` / `自动更新HTML.bat`：监听排期表变化，自动重新生成 HTML。

## 排期表基础填写

1. 第 2 行是日期行，格式必须类似 `6月15日`。当前年份固定为 `2026`。
2. 每个日期列代表“活动开启日期”。
3. 从第 3 行开始，日期列里只要有内容，就会被当成一个活动。
4. 一个格子里可以写多行。脚本会保留原始内容，用来识别活动名、服务器、礼包 ID 等。
5. 这些纯备注会被忽略，不生成提醒：
   - `兑换`
   - `兑换多一天`
   - `礼包【✔】+分组预设【✔】+活动确认【✔】`
   - `活动预设【✔】+公布分组【✔】`
   - `这轮不开，等0602版本改竞技场规则`

## 活动 ID / 展示名规则

展示名优先级：

1. `进化连冲` 固定展示 4 个活动：
   - `1900133: 连续充值（进化开启第二套）`
   - `1900162: 连续充值（进化开启第三套）`
   - `1900163: 连续充值（改造，开启进化第2/3套以外的服务器）`
   - `1900084:【国服】五日随心储`
2. `活动与礼包对应关系.xlsx` 里，活动名能匹配且 `常规活动` 列为 `1`，会用映射表里的活动 ID 文本展示。
3. 即使 `常规活动` 不是 `1`，这些活动也会强制使用映射表：`落潮海岸`、`黑骑士`、`珍宝商店`、`自选周卡`、`特化蚁对决`、`蚁后对决`。
4. 如果排期格子第一行本身是 `数字: 名称`，且没有命中映射，会直接显示这一行。
5. 都没有命中时，显示识别出来的活动名。

新增常规活动时，推荐在 `活动与礼包对应关系.xlsx` 增加一行：

```text
活动和礼包的对应关系 = 活动名
活动id = 要展示的 ID 和名称
常规活动 = 1
```

多活动 ID 可以在映射表同一个单元格里换行。

`进化连冲` 当前服务器展示格式：

```text
1900133: 连续充值（进化开启第二套）
```

其中 `S854` 会按每期 +2 自动变化。

### 从排期文本解析服务器

自动规则没命中时，按下面顺序解析：

1. 写了 `服务器：全服`，显示 `全服`。
2. 功勋商店文本里写了数字或 `xxx`，显示成 `S数字` 或 `Sxxx`。
3. 黑骑士会识别 `客户端 S...` 和 `小程序 S...`，分别展示。
4. 识别 `S1-S854`、`S1~S854`、`S854`、`Sxxx` 这类服务器文本。
5. 识别 `1~854`、`1～xxx` 这类范围，并规范成 `S1-S854` / `S1-Sxxx`。
6. 识别 `【...1~854...】` 这类括号内范围。
7. 都没有识别到时，显示 `排期表未标明`。

如果某个新活动没有自动服务器规则，建议在排期格子里明确写：

```text
服务器：全服
```

或：

```text
S1-S999
```

或：

```text
1~999
```

## 新增活动时怎么判断要改哪里

只新增一条普通排期：

- 直接把活动内容填到对应开启日期列。
- 如果服务器不能自动算，在格子里写清楚服务器范围。
- 如果展示名需要固定活动 ID，优先改 `活动与礼包对应关系.xlsx`。

新增一个现有周期活动的新日期：

- 只填排期即可。
- 例如 `进化连冲 / 新野怪连储 / 新变异连储` 继续按 14 天轮换，服务器尾号自动 +2。

新增一个全新的规则：

- 如果提醒日期不同，比如提前 3 天、周一提醒，需要改 `reminder_rules()`。
- 如果持续时间不同，需要改 `DURATIONS`。
- 如果服务器要自动推算，需要改 `computed_server_text()`。
- 如果 HTML 要展示特殊礼包块，需要改 `package_details()`。

## 当前重要限制

- 年份固定是 `2026`。如果排期跨到 2027，需要改 `YEAR`。
- 自动服务器规则都依赖固定基准日期。排期日期如果不在周期上，会退回文本解析。
- 活动名识别靠关键词，所以新活动名称里最好保留当前关键词，例如 `进化连冲`、`新野怪连储`、`新变异连储`。
